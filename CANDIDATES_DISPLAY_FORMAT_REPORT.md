# Отчет об изменении формата отображения кандидатов

## 🎯 Проблема

При поиске кандидатов в Telegram боте отображался только:
```
Пользователь (совпадение: 15%)
```

Не хватало важной информации для принятия решения о выборе партнера.

## 🔧 Решение

Изменен формат отображения кандидатов на более информативный:
```
Имя - Выбранная тема - Предпочитаемое время (совпадение: 15%)
```

## 📝 Внесенные изменения

### Обновлен метод `_handle_find_callback` в `orator_callback_handler.py`

#### Старый код:
```python
for candidate in candidates[:5]:
    name = candidate.get("first_name", "Пользователь")
    score = candidate.get("match_score", 0)
    keyboard.append([
        InlineKeyboardButton(
            f"{name} (совпадение: {score:.1%})", 
            callback_data=f"candidate_{candidate['user_id']}"
        )
    ])
```

#### Новый код:
```python
for candidate in candidates[:5]:
    name = candidate.get("name", "Пользователь")
    score = candidate.get("match_score", 0)
    preferred_time = candidate.get("preferred_time_msk", "Не указано")
    selected_topics = candidate.get("selected_topics", [])
    
    # Берем первую тему или показываем "Не выбрано"
    topic_display = selected_topics[0] if selected_topics else "Не выбрано"
    
    # Формируем текст кнопки в новом формате
    button_text = f"{name} - {topic_display} - {preferred_time} (совпадение: {score:.1%})"
    
    keyboard.append([
        InlineKeyboardButton(button_text, callback_data=f"candidate_{candidate['user_id']}")
    ])
```

## 🎨 Новый формат отображения

### Примеры отображения кандидатов:

**Старый формат:**
```
Пользователь (совпадение: 56.5%)
Пользователь (совпадение: 53.0%)
Пользователь (совпадение: 44.5%)
```

**Новый формат:**
```
Анна Иванова - Группа 1 - Уровень 1 - 19:00 (совпадение: 56.5%)
Елена Сидорова - Группа 2 - Уровень 1 - 19:30 (совпадение: 53.0%)
Михаил Петров - Группа 1 - Уровень 2 - 20:00 (совпадение: 44.5%)
```

## 📊 Отображаемая информация

1. **Имя кандидата** - полное имя пользователя
2. **Выбранная тема** - первая из выбранных тем (или "Не выбрано")
3. **Предпочитаемое время** - время в формате ЧЧ:ММ (или "Не указано")
4. **Процент совпадения** - рассчитанный алгоритмом матчинга score

## 🔍 Логика обработки данных

### Обработка имени:
- Используется поле `name` из API ответа
- Fallback: "Пользователь" если имя не указано

### Обработка тем:
- Берется первая тема из массива `selected_topics`
- Fallback: "Не выбрано" если темы не указаны

### Обработка времени:
- Используется поле `preferred_time_msk`
- Fallback: "Не указано" если время не указано

### Обработка score:
- Форматируется как процент с одним знаком после запятой
- Пример: 0.565 → 56.5%
